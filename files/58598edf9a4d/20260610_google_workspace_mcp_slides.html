01
Google Workspace を
データ基盤にする
AIを活かすには、まずデータが「整っている」こと

なぜ、データはあるのに活かせないのか

税理士事務所の現実
📊 会計ソフトの数字 → 会計ソフトの中
📄 書類・領収書 → バラバラのフォルダ
💬 面談の記録 → 議事録ファイル
📧 メールの履歴 → Outlookの受信箱
📅 予定・期限 → 担当者の手帳
顧客の全体像を把握するために、毎回いくつもの画面を開いて頭の中で繋ぎ合わせる
理想の状態
📁 顧客フォルダを開けば
　→ 数字も書類も議事録も全部ここにある

🤖 AIは「どこを見ればいいか」を知っている
　→ 日本語で聞くだけで答えが返る
「データ量」の問題ではない。顧客単位でデータが揃っていないことが問題

なぜ Google Workspace がデータ基盤として優れているか

1
クラウドネイティブ

全データがクラウド上に存在。AIがAPIで直接アクセス・読み書きできる。ローカルにファイルを落とす手間がゼロ

2
リアルタイム性

Gmail・カレンダー・Driveは常に最新状態。「昨日のメール」は今日取得しても必ず最新が返ってくる

3
検索性の高さ

Google検索と同じエンジン。件名・本文・PDF内テキストを横断検索。AIの「情報探し」コストが激減する

4
公式MCP・API連携

Claude CodeとのMCP連携が公式に整備済み。Gmail・Drive・Calendarをまとめて操作できる

Google Workspaceにデータが揃った後、記帳はTaxis・それ以外はMCP+Claude Codeで自動化できる

顧客フォルダの設計が「土台」になる

TaxSysは顧問先を登録するだけで、Google Drive上に10フォルダ構成を自動生成する

02
Google MCP × Claude Code
Taxisで対応できない領域を、AIが自律的に自動化する

MCP（Model Context Protocol）とは

MCPなし（従来）
🤖 ✗ 📁
ファイルを手動でダウンロード → AIに貼り付け
→ 結果を手動でDriveに入力
「AIが考えるだけ。操作は人間」
MCPあり（今日の話）
🤖
Claude Code
⇄
MCP
📧📁📅
Google Workspace
Gmailを読む → 情報を整理 → Driveに保存 → Calendarに登録
Claude が自律的に完結させる
MCPは「AIとツールを繋ぐ標準プロトコル」。Anthropicが2024年11月に公開。記帳以外の自動化はすべてここで完結する

Google MCP でできること

Gmail
  • 受信メールの分類・抽出
  • 返信・下書き作成
  • 顧問先へのリマインド一括送信
  • メール情報をNotionへ転記
Google Drive
  • Drive内PDFの読み取り・要約
  • Sheetsへのデータ書き込み
  • Docのドキュメント生成・編集
  • 会計ソフトへの連携データ作成
Google Calendar
  • 面談予定の検索・作成
  • 空き時間の自動確認
  • MTG後タスク自動登録
  • 申告期限アラート管理
これらを 「日本語で指示するだけ」 で自動実行。コードを書く必要はゼロ
